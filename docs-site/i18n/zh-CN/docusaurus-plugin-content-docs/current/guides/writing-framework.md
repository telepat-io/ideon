# 写作框架

Ideon 现已采用“指南优先”的提示词组合方式。写作行为来自 `writing-guide/` 下的 markdown 指南文件，并按阶段组合。

## 核心原则

1. 结构服务意图
2. 信息密度优先于填充内容
3. 具体优先于模糊
4. 节奏与可读性并重
5. 可扫描性与结构化路标
6. 主动语态与具体主语
7. 有纪律的叙事
8. 渠道原生表达
9. 真实性过滤（朴素、直接的语言）

## 建议与避免

建议：

- 使用具体机制与示例。
- 用清晰钩子开篇。
- 从开头到结尾保持明确主线。
- 混合短句、中句和长句形成自然节奏。
- 以有信息量的陈述句开启段落。
- 优先可度量或可操作验证的表达。
- 根据渠道预期调整结构。

避免：

- 无证据的泛化结论。
- 重复单一的句式节奏。
- 营销腔填充和夸张措辞。
- 不增加信息量的空洞总结句。
- 过度打磨、AI 味重的过渡与戏剧化陈词滥调。
- 生搬硬套文章结构到社交平台格式。

## 风格选择

你可以设置一个运行级风格。Ideon 会用该值选择 `writing-guide/styles/` 下对应的风格指南文件。

- `academic`
- `analytical`
- `authoritative`
- `conversational`
- `empathetic`
- `friendly`
- `journalistic`
- `minimalist`
- `persuasive`
- `playful`
- `professional`
- `storytelling`
- `technical`

## 提示词组合模型

Ideon 会将“阶段指南 + 运行元数据”组合成提示词：

1. 阶段指南集合（规划、正文写作、共享简报、渠道改写）
2. 选定风格指南（`writing-guide/styles/<style>.md`）
3. 选定意图指南（`writing-guide/content-intent/<intent>.md`）
4. 选定格式指南（`writing-guide/formats/<content-type>.md`）
5. 代码中的运行约束（运行上下文与目标长度）

这种方式将写作规则沉淀到可版本化的指南文件，同时保留可预测的运行时约束。

## 在多输出运行中的应用

- 同一套风格与意图指南会应用到该次运行的所有输出。
- 随后由格式指南对不同渠道进行结构特化。
- 当包含文章输出时，社交输出可将文章作为锚定上下文。

实践含义：

- 每次运行仅用一种风格以保证一致性，再调整目标类型和数量。
- 若需要差异很大的语气，请拆分为多次运行。
